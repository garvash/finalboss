## Implementation Patterns

- **Naming Patterns:**
    - Node Name: `PascalCase` (例: `SummonButton`)
    - Function/Variable: `snake_case` (例: `current_gold`, `_on_unit_reached`)
- **Signal Patterns:**
    - シグナル接続は `_ready()` 内のコードで行う。
    - ハブとなる `GlobalSignals` (Autoload) を経由させる。
- **Conflict Logic:**
    - 勇者とモンスターが `Area2D` で重なった際、`speed` を `0` にし、`Timer` を開始する。
    - `Timer` 終了後に相手の `health` を減らし、死ななければ移動を再開する。

## Data Architecture
### GameManager (Autoload)
- `var gold: int = 100`
- `var luck_meter: float = 0.0`
- `var survival_time: float = 0.0`
- `func add_gold(amount: int)`: `GlobalSignals.gold_changed` を発火。

## Performance Considerations
- **Draw Calls:** Kenneyアセットを同一の `TextureAtlas` にまとめることでドローコールを抑制。
- **KISS Principles:** 1画面固定（Fixed Camera）により、カリングや複雑な視錐台計算を排除。

---

## Success Metrics
- **KISS Check:** 新機能追加の際、このアーキテクチャの範疇を超えないか（1時間以内に実装できるか）を常に問う。
